API Documentation

URL: /api/account
Supported Methods: GET
Middleware: None
Query/Body Params: None
Description: Checks the current session and returns the logged-in account if one exists. Returns an error if the user is not logged in.
Return Type(s): JSON

URL: /api/login
Supported Methods: POST
Middleware: Requires Secure, Requires Logout
Query/Body Params: username (required account username), password (required account password)
Description: Authenticates a user with their username and password. If successful, saves the account information into the session.
Return Type(s): JSON

URL: /api/logout
Supported Methods: POST
Middleware: Requires Login
Query/Body Params: None
Description: Destroys the current user's session and logs the user out.
Return Type(s): Status code

URL: /api/signup
Supported Methods: POST
Middleware: Requires Secure, Requires Logout
Query/Body Params: username (required new account username), pass (required new account password)
Description: Creates a new account, hashes the password, saves the account in MongoDB, and logs the new user in.
Return Type(s): JSON

URL: /api/resetpass
Supported Methods: POST
Middleware: Requires Login
Query/Body Params: username (required username for the account), pass (required new password)
Description: Resets the password for the provided username by hashing the new password and saving it to the account.
Return Type(s): JSON

URL: /api/posts
Supported Methods: GET
Middleware: None
Query/Body Params: None
Description: Retrieves all posts sorted by newest first. If a post is locked Onlyfans content, the response hides the content unless the current user owns or has paid for the post.
Return Type(s): JSON

URL: /api/users/:username/posts
Supported Methods: GET
Middleware: None
Query/Body Params: username (path parameter for the user whose posts are being requested)
Description: Retrieves posts created by a specific username. Returns 404 if the username does not exist.
Return Type(s): JSON

URL: /api/posts
Supported Methods: POST
Middleware: Requires Login
Query/Body Params: content (required post text), onlyfans (optional boolean for locked paid content)
Description: Creates a new post owned by the logged-in user.
Return Type(s): JSON

URL: /api/posts/:id/pay
Supported Methods: POST
Middleware: Requires Login
Query/Body Params: id (path parameter for the post being paid for)
Description: Adds the logged-in user to the post's fans list so they can access locked Onlyfans content.
Return Type(s): JSON

URL: /api/posts/:id
Supported Methods: DELETE
Middleware: Requires Login
Query/Body Params: id (path parameter for the post being deleted)
Description: Deletes a post if it exists and is owned by the logged-in user.
Return Type(s): JSON
